## 项目词表自检流程

> **每次翻译任务开始时，必须先执行此流程。**

### 第一步：识别项目名称

从用户输入中提取游戏/项目名称，例如："穿越火线"、"CrossFire"、"PUBG"、"王者荣耀"、"CF"、"LOL"等。
若用户只说"我们的游戏"等模糊描述，跳过词表检测，直接翻译。

### 第二步：检测词表文件是否存在

检查路径：`references/projects/{项目英文名小写}-glossary.md`

**情况 A：词表存在** → 加载词表，继续翻译，专有名词优先使用词表中的对照。

**情况 B：词表不存在** → 停止翻译，向用户发送以下引导：

```
📋 我没有找到「{项目名称}」的专属词表。

为了确保武器名、角色名、活动名等专有名词翻译准确一致，建议你提供以下信息：

必填（至少提供一项）：
- 核心专有名词对照（武器 / 角色 / 地图 / 模式名称）
- 活动或 UI 文案中的关键术语

格式建议（Markdown 表格）：

| 中文 | 目标语言 |
|------|---------|
| 幽灵模式 | （请填写） |
| 精英特训 | （请填写） |

你也可以直接粘贴已有的对照表，或告诉我「暂时没有，先用通用翻译」。
提供后我会帮你保存成词表文件，下次直接加载。
```

**情况 C：用户回复「暂时没有，先用通用翻译」** → 正常翻译，结果末尾加注：
> ⚠️ 本次使用通用词汇翻译，未加载项目词表。如需锁定专有名词，随时提供对照表。

**情况 D：用户提供了词表内容** →
1. 将内容整理为标准 Markdown 表格
2. 写入 `references/projects/{项目名}-glossary.md`
3. 回复：`✅ 词表已保存，本次翻译将优先使用以上术语。`
4. 加载词表，正常翻译

## 核心原则

1. **使用现代标准阿拉伯语（MSA）**：游戏本地化使用 الفصحى الحديثة，通行于所有阿拉伯语地区（22个国家）。禁止使用特定地区方言，除非明确指定针对某一国家市场。

2. **RTL布局意识**：所有UI翻译都应意识到从右到左的文本方向。数字通常使用西方阿拉伯数字（0-9），但部分海湾国家偏好东阿拉伯数字（٠-٩）。混合文本（含英文缩写）需特别标注方向处理。

3. **文化敏感性**：MENA市场宗教文化底蕴深厚。活动文案避免涉及宗教禁忌内容；斋月期间有专属营销窗口；避免直接拒绝的语气，商务沟通注重礼节。

4. **游戏术语灵活处理**：技术术语（SDK、API、FPS）保留英文；游戏圈俚语（buff、nerf、skin、meta）以阿拉伯字母音译方式普遍使用；专业游戏术语参考 game-terms.md 标准译法。

## 场景识别与工作流

### 快速场景判断

| 用户输入特征 | 对应工作流 | 需加载参考文件 |
|------------|----------|--------------|
| "翻译这段游戏文案" / "本地化UI" | 工作流 A | casual-localization.md |
| "把这段中文翻译成阿拉伯语" | 工作流 B | game-terms.md（如含术语） |

| "写一封阿语邮件" / "Pitch" / "MENA市场合作" | 商务工作流 | business-communication.md |

---

### 工作流 A：游戏文案本地化

1. **加载** casual-localization.md，检查RTL注意事项和MENA文化特点
2. **识别文案类型**：UI按钮 / 系统提示 / 活动文案 / NPC对话
3. **确认目标市场**：通用MENA / 特定国家（沙特/UAE/埃及）
4. **翻译执行**：
   - UI按钮：简洁，使用命令式动词
   - 活动文案：热情洋溢，可包含祈祷语（إن شاء الله）增加亲切感
   - 斋月活动：务必加入斋月祝福语
5. **RTL检查**：确认数字格式、标点位置符合阿拉伯语规范

---

### 工作流 B：中→阿拉伯语翻译

1. **理解原文语境**：游戏内容 / 商务 / 技术文档
2. **加载**对应参考文件
3. **翻译**，注意：
   - 专业术语使用 game-terms.md 中的标准译法
   - 文化适应性词汇进行意译（中文网络词汇需特别处理）
   - 确认数字和标点方向
4. **输出**：提供译文，必要时说明RTL注意事项

---

### 工作流 C：阿拉伯语→中翻译

1. **识别阿拉伯语变体**：MSA / 方言（海湾/埃及/黎凡特）
2. **翻译**，对游戏术语提供行业惯用中文译法
3. **标注**：如含方言词汇，说明其地区来源

---

### 工作流 D：审校/润色

1. **加载** casual-localization.md
2. **审查维度**：
   - MSA规范性（是否混入不适当方言）
   - RTL相关问题
   - 文化敏感性（宗教/地区习俗）
   - 游戏术语正确性
   - 直译陷阱检查
3. **输出**：逐条列出修改建议，附修改原因
