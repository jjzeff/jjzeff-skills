# 日语商务沟通指南（游戏行业）

## 节1：日语商务语气指南

### 1.1 日本商务文化基础

**礼节第一**
- 日本商务文化极重视礼节（礼儀、れいぎ）和形式
- 邮件开头必须有固定问候语，结尾必须有署名（名前と連絡先）
- 不能太直接——委婉表达（遠回しな表現）是常态
- 拒绝时用「前向きに検討いたします」（积极考虑）暗示，很少直说"NO"

**头衔极为重要（役職は非常に重要）**
- 称呼对方时必须用：`[姓名] + [役職] + 様`
- 例：「田中社長様」「山田部長様」「鈴木様」（无头衔时）
- 邮件宛名格式：
  ```
  ○○株式会社
  ○○部 ○○様
  ```
- 自称时用「私（わたくし）」（正式）而非「私（わたし）」

**敬語层级速查**

| 场景 | 使用敬語 | 示例 |
|------|----------|------|
| 对对方公司行为 | 尊敬語 | 「ご確認いただけますでしょうか」 |
| 对自己公司行为 | 謙譲語 | 「ご送付いたします」「ご連絡申し上げます」 |
| 一般礼貌 | 丁寧語 | 「～です」「～ます」 |

### 1.2 商务邮件套語（必须掌握）

| 用途 | 日語套語 |
|------|---------|
| 邮件开头（常用往来） | お世話になっております。（おせわになっております） |
| 邮件开头（首次联系） | 突然のご連絡、失礼いたします。 |
| 邮件开头（回复） | ご返信ありがとうございます。 |
| 引出话题 | 早速ですが、本日は～についてご連絡いたしました。 |
| 请对方确认 | ご確認のほど、よろしくお願いいたします。 |
| 请对方回复 | ご返信のほど、よろしくお願い申し上げます。 |
| 附件说明 | 添付ファイルをご確認ください。 |
| 期待合作 | 引き続きよろしくお願いいたします。 |
| 有事相告 | 取り急ぎご連絡いたします。 |
| 道歉 | 誠に申し訳ございません。ご迷惑をおかけし、大変失礼いたしました。 |

## 节3：会议常用语（日语）

### 3.1 開始の挨拶

```
「本日はお忙しい中、お時間をいただきありがとうございます。」
（感谢大家百忙之中抽出时间。）

「それでは、時間になりましたので始めさせていただきます。」
（时间到了，我们开始吧。）

「本日の司会を務めます、○○と申します。よろしくお願いします。」
（今天由我○○主持会议，请多关照。）

「まず、簡単に自己紹介をさせていただければと思います。」
（首先，请允许我做一个简单的自我介绍。）
```

### 3.2 質問（提问）

```
「一点確認させていただいてもよろしいでしょうか。」
（我可以确认一点吗？）

「○○についてもう少し詳しくお聞かせいただけますか。」
（能否就○○再详细说明一下？）

「すみません、少しよろしいでしょうか。○○という部分なのですが……」
（不好意思，我有个疑问，关于○○这个部分……）

「ご質問いただきありがとうございます。ご説明いたします。」
（感谢您的提问，让我来说明一下。）
```

### 3.3 進行（会议推进）

```
「では、次の議題に移らせていただきます。」
（那么，我们进入下一个议题。）

「お手元の資料○ページをご覧ください。」
（请看您手边资料的第○页。）

「ここまでで、ご質問はありますでしょうか。」
（到这里为止，请问有什么问题吗？）

「少々お時間をいただいてよろしいでしょうか。確認してまいります。」
（能稍微给我一点时间吗？我去确认一下。）

「大変申し訳ありませんが、予定の時間が迫っております。」
（非常抱歉，预定时间已经快到了。）
```

### 3.4 締め括り（结束）

```
「本日はありがとうございました。大変参考になりました。」
（今天非常感谢，受益良多。）

「後ほど議事録をお送りいたします。」
（稍后我会发送会议纪要。）

「次回は○月○日にお話しできればと思います。」
（希望○月○日能再次与您交流。）

「引き続きよろしくお願いいたします。それでは失礼いたします。」
（请继续关照。那我们就到这里，再见。）
```

## 节4：Pitch提案结构 + データ提示例文（日语）

### 4.1 Pitch提案推荐结构

```
1. タイトル／表紙（封面）
   - ゲームタイトル ＋ ジャンル ＋ ターゲット市場 ＋ 会社名

2. エグゼクティブサマリー（Executive Summary）
   - 1スライドでゲームと提案の要点を整理

3. 市場機会（Market Opportunity）
   - 日本・アジアのゲーム市場規模と成長トレンド

4. ゲーム概要（Game Overview）
   - ゲームプレイ動画 ＋ 特徴（コアループ）＋ スクリーンショット

5. なぜ今？なぜ弊社？（Why Now? Why Us?）
   - チームの実績、既存タイトルのKPI

6. ビジネスモデル（Business Model）
   - 収益化方法（F2P ＋ ガチャ ／ 買い切り）
   - 目標DAU、ARPU、LTV

7. パートナーシップ提案（Partnership Proposal）
   - 希望するパブリッシャーの役割
   - レベニューシェア案

8. ロードマップ（Roadmap）
   - 開発状況とリリース予定

9. 次のステップ（Next Steps）
   - NDA → デモ → 契約交渉
```

```

「弊社の既存タイトルでは、リリース後30日のリテンション率が○○%を達成し、
```

```
　・Day7リテンション：○○%
```

```
　・マーケティング費用：双方折半
　・契約期間：初期○年間、その後更新オプション付き
 
 詳細条件はNDA締結後、改めてご説明できれば幸いです。」
```
