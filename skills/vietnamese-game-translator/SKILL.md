## 项目词表自检流程

> **每次翻译任务开始时，必须先执行此流程。**

### 第一步：识别项目名称

从用户输入中提取游戏/项目名称，例如："穿越火线"、"CrossFire"、"PUBG"、"王者荣耀"、"CF"、"LOL"等。
若用户只说"我们的游戏"等模糊描述，跳过词表检测，直接翻译。

### 第二步：检测词表文件是否存在

检查路径：`references/projects/{项目英文名小写}-glossary.md`

**情况 A：词表存在** → 加载词表，继续翻译，专有名词优先使用词表中的对照。

**情况 B：词表不存在** → 停止翻译，向用户发送以下引导：

```
📋 我没有找到「{项目名称}」的专属词表。

为了确保武器名、角色名、活动名等专有名词翻译准确一致，建议你提供以下信息：

必填（至少提供一项）：
- 核心专有名词对照（武器 / 角色 / 地图 / 模式名称）
- 活动或 UI 文案中的关键术语

格式建议（Markdown 表格）：

| 中文 | 目标语言 |
|------|---------|
| 幽灵模式 | （请填写） |
| 精英特训 | （请填写） |

你也可以直接粘贴已有的对照表，或告诉我「暂时没有，先用通用翻译」。
提供后我会帮你保存成词表文件，下次直接加载。
```

**情况 C：用户回复「暂时没有，先用通用翻译」** → 正常翻译，结果末尾加注：
> ⚠️ 本次使用通用词汇翻译，未加载项目词表。如需锁定专有名词，随时提供对照表。

**情况 D：用户提供了词表内容** →
1. 将内容整理为标准 Markdown 表格
2. 写入 `references/projects/{项目名}-glossary.md`
3. 回复：`✅ 词表已保存，本次翻译将优先使用以上术语。`
4. 加载词表，正常翻译

## 核心原则

- **声调符号必须完整**：越南语6个声调，漏掉声调符号不仅不专业，还会改变词义（如 ma/má/mà/mả/mã/mạ意思完全不同）
- **人称代词选择**：游戏内推荐 bạn（友好中性）；商务用 Quý vị 或对方头衔；避免太随意的 mày/tao
- **南北差异注意**：标准使用北方语（Hà Nội），对话/活动文案可适当融入南方口语增加亲近感
- **英文借词自然**：skill/boss/buff/nerf/lag等越南游戏圈直接用，无需翻译

## 场景识别与工作流

### 快速场景判断
1. 游戏文案/本地化 → 先读 casual-localization.md
2. 商务沟通 → 先读 business-communication.md + game-terms.md
3. 专业术语 → 读 game-terms.md
4. 混合场景 → 同时读相关文件

### 工作流 A：游戏文案本地化
1. 判断文案类型（UI/活动/剧情/推送/社区）
2. 读 casual-localization.md
3. 检查：声调符号完整、bạn称谓一致、英文借词是否自然融合
4. 输出：原文 + 译文 + 说明

### 工作流 B：中→越南语翻译
1. 货币：VND/đồng(₫)；日期：DD/MM/YYYY
2. 所有声调符号必须准确
3. 输出：原文 + 译文

### 工作流 C：越南语→中翻译
1. 注意声调区分同音字
2. 南方口语识别

### 工作流 D：审校/润色
1. 检查声调符号
2. 检查人称代词一致性
3. 检查南北方言一致性
4. 给出修改建议
