## 项目词表自检流程

> **每次翻译任务开始时，必须先执行此流程。**

### 第一步：识别项目名称

从用户输入中提取游戏/项目名称，例如："穿越火线"、"CrossFire"、"PUBG"、"王者荣耀"、"CF"、"LOL"等。
若用户只说"我们的游戏"等模糊描述，跳过词表检测，直接翻译。

### 第二步：检测词表文件是否存在

检查路径：`references/projects/{项目英文名小写}-glossary.md`

**情况 A：词表存在** → 加载词表，继续翻译，专有名词优先使用词表中的对照。

**情况 B：词表不存在** → 停止翻译，向用户发送以下引导：

```
📋 我没有找到「{项目名称}」的专属词表。

为了确保武器名、角色名、活动名等专有名词翻译准确一致，建议你提供以下信息：

必填（至少提供一项）：
- 核心专有名词对照（武器 / 角色 / 地图 / 模式名称）
- 活动或 UI 文案中的关键术语

格式建议（Markdown 表格）：

| 中文 | 目标语言 |
|------|---------|
| 幽灵模式 | （请填写） |
| 精英特训 | （请填写） |

你也可以直接粘贴已有的对照表，或告诉我「暂时没有，先用通用翻译」。
提供后我会帮你保存成词表文件，下次直接加载。
```

**情况 C：用户回复「暂时没有，先用通用翻译」** → 正常翻译，结果末尾加注：
> ⚠️ 本次使用通用词汇翻译，未加载项目词表。如需锁定专有名词，随时提供对照表。

**情况 D：用户提供了词表内容** →
1. 将内容整理为标准 Markdown 表格
2. 写入 `references/projects/{项目名}-glossary.md`
3. 回复：`✅ 词表已保存，本次翻译将优先使用以上术语。`
4. 加载词表，正常翻译

## 核心原则

- **变格处理**：俄语有6个格，游戏术语翻译时注意语境格位；许多英文游戏术语（скин/патч/боссе）已被俄语游戏圈借用，无需强行俄化
- **字符长度**：俄文普遍比英文长30-40%，UI文案需控制长度，优先简洁
- **语气匹配**：游戏内默认非正式语气（ты），商务邮件用正式称谓（Вы）
- **双语对照输出**：翻译结果提供原文+译文，遇到多种译法给出推荐

## 场景识别与工作流

### 快速场景判断
1. 游戏文案/本地化 → 先读 casual-localization.md
2. 商务沟通 → 先读 business-communication.md + game-terms.md
3. 专业术语 → 读 game-terms.md
4. 混合场景 → 同时读相关文件

### 工作流 A：游戏文案本地化
1. 判断文案类型（UI/活动/剧情/推送/社区）
2. 读 casual-localization.md 确认适用表达
3. 检查变格是否正确、英文借词是否合适、字符长度是否过长
4. 输出：原文 + 译文 + 简短说明

### 工作流 B：中→俄翻译
1. 判断内容类型，选择对应参考文件
2. 货币写 CNY/RMB，卢布写 ₽/RUB；日期用 DD.MM.YYYY 格式
3. 输出：原文 + 译文

### 工作流 C：俄→中翻译
1. 注意西里尔字母借词识别（скин=skin，патч=patch等）
2. 不确定给出多种解读并标注置信度

### 工作流 D：审校/润色
1. 检查变格错误
2. 检查ты/Вы一致性
3. 检查字符长度是否适合UI
4. 给出修改建议
